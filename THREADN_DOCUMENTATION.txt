                           1. PROJECT OVERVIEW
================================================================================

WHAT IS THREADN?
----------------
ThreadN is an AI-powered SaaS platform that transforms any content - ideas,
YouTube videos, transcripts, or topics - into viral social media content. It
generates 10 scored viral hooks with evidence-based reasoning, then builds
complete threads ready for multi-platform posting.

TARGET AUDIENCE:
- Content Creators
- Social Media Managers
- Digital Marketers
- YouTubers & Podcasters
- Entrepreneurs & Coaches
- Freelance Writers
- Marketing Agencies

CORE VALUE PROPOSITION:
"Stop struggling to write. Start creating viral content."

KEY BENEFITS:
- Transform 1 piece of content into 10 viral hooks in seconds
- Auto-extract transcripts from any YouTube video
- Virality scores (0-5) with evidence from your content
- One-click thread generation with custom CTAs
- Format for any platform (Facebook, Twitter, LinkedIn, Instagram)
- Schedule, organize, and track your content

================================================================================
                       4. CONTENT GENERATION SYSTEM
================================================================================

4.1 HOOK GENERATION PROCESS
---------------------------

INPUT:
- User provides transcript/content (any length)
- System sanitizes and wraps input

PROCESSING:
- GPT-4o-mini analyzes content
- Identifies key insights and patterns
- Applies viral hook frameworks:
  * Curiosity gaps
  * Controversy/hot takes
  * Relatability moments
  * Pattern interrupts
  * Emotional triggers

OUTPUT FORMAT:
{
  "hooks": [
    {
      "hook": "The text of the viral hook (max 15 words)",
      "score": 4.5,
      "reason": "Why this hook works and drives engagement",
      "evidence": "Direct quote from transcript supporting this hook"
    },
    // ... 9 more hooks
  ]
}

SCORING CRITERIA (0-5):
- 5: Extremely viral potential, broad appeal, strong emotion
- 4: High engagement, proven pattern, clear value
- 3: Good hook, solid engagement expected
- 2: Average, may work with right audience
- 1: Weak, needs improvement
- 0: Not recommended

4.2 THREAD GENERATION PROCESS
-----------------------------

INPUT:
- Selected hook (from generation)
- Original transcript
- Custom CTA (optional)

PROCESSING:
- Expands hook into full narrative
- Creates logical flow
- Adds supporting points from content
- Integrates engaging elements

OUTPUT FORMAT:
{
  "title": "Thread title based on hook",
  "body": "
    [Hook text]

    1. First point with emoji
    2. Second point with supporting detail
    3. Third point with insight
    4. Fourth point with example
    5. Fifth point with takeaway
    6. Sixth point (if applicable)
    7. Seventh point (if applicable)

    [Custom CTA or default]
  "
}

4.3 YOUTUBE TRANSCRIPT EXTRACTION
---------------------------------

5 FALLBACK METHODS (in order):

1. youtubei.js Library
   - Uses Innertube API
   - Gets captions directly from YouTube
   - Highest quality transcripts

2. Direct Caption Track Fetch
   - Fetches caption tracks from YouTube
   - Parses XML/JSON responses
   - Works for most public videos

3. Timedtext API
   - Legacy YouTube caption API
   - Fallback for older videos
   - Supports multiple languages

4. Video Description
   - Extracts description text
   - Useful when captions disabled
   - Last resort for content

5. oEmbed API
   - Gets video title at minimum
   - Ensures something is always returned
   - Fallback of last resort

--------------

- email (unique, lowercase)
- password (hashed, hidden from queries)

- authProvider ("credentials" | "google")
- googleId (for OAuth users)
- role ("user" | "admin")
- plan ("free" | "starter" | "pro" | "enterprise")
- coins (for extra generations)
- subscription {
    status: "active" | "cancelled" | "expired"
    plan: string
    expiresAt: Date
    paddleSubscriptionId: string
    paddleCustomerId: string
  }
- usage {
    totalHooks: number
    totalThreads: number
    dailyThreads: number
    lastThreadDate: Date
    dailyCost: number
    monthlyCost: number
    lastCostReset: Date
  }

6.2 GENERATION MODEL
--------------------
Fields:
- userId (reference to User)
- type ("hooks" | "thread")
- input {
    transcript: string
    cta: string (for threads)
    selectedHook: string (for threads)
  }
- output {
    hooks: Array (for hook generations)
    thread: { title, body } (for threads)
  }
- tokens (usage count)
- cost (in USD)
- timestamps

6.3 ACCESS CODE MODEL
---------------------
Fields:
- code (unique, uppercase)
- type ("subscription" | "coins" | "trial")
- value (days for subscription, count for coins)
- plan (for subscription codes)
- maxUses, usedCount
- usedBy (array of userIds)
- isActive
- expiresAt
- createdBy (admin userId)

6.4 SCHEDULED POST MODEL
------------------------
Fields:
- userId
- content {
    hook: string
    thread: string
    platforms: string[]
  }
- scheduledAt (Date)
- status ("pending" | "published" | "failed")
- recurrence ("none" | "daily" | "weekly")
- publishedAt, error
- timestamps

6.5 TEMPLATE MODEL
------------------
Fields:
- userId (null for system templates)

- content {
    hookTemplate: string

  }

- timestamps

6.6 COLLECTION MODEL
--------------------
Fields:
- userId

- timestamps

6.7 FAVORITE MODEL

Fields:

6.8 USER STREAK MODEL
---------------------

- currentStreak

- totalDaysActive

6.9 USER ACHIEVEMENT MODEL
